mysql数据库

发表于 2018-05-19 | 分类于 数据库

前言：

MySQL是一个关系型数据库管理系统，由瑞典MySQL AB 公司开发，目前属于 Oracle 旗下产品。MySQL 是最流行的关系型数据库管理系统之一，在 WEB 应用方面，MySQL是最好的 RDBMS (Relational Database Management System，关系数据库管理系统) 应用软件。

MySQL是一种关系数据库管理系统，关系数据库将数据保存在不同的表中，而不是将所有数据放在一个大仓库内，这样就增加了速度并提高了灵活性。

MySQL所使用的 SQL 语言是用于访问数据库的最常用标准化语言。MySQL 软件采用了双授权政策，分为社区版和商业版，由于其体积小、速度快、总体拥有成本低，尤其是开放源码这一特点，一般中小型网站的开发都选择 MySQL 作为网站数据库。

数据库的基本操作

数据库的操作

语法：

创建数据库：
create database 数据库名称；
creat database 数据库名称 character set 编码 collate 校对规则；
例：创建一个使用utf8字符集，并带校对规则的mydb数据库
create database mydb character set 'UTF-8' collate 'utf8_bin' 

查看数据库：
show databases；（查看所有建立的数据库）
show create database mydb（查看名为mydb的数据库）

删除数据库：
drop database mydb（删除名为mydb的数据库）

修改数据库：
alter database mydb character set 'gbk' （修改数据库mydb的编码）

切换数据库：也就是使用该数据库
use mydb；

表中数据的类型与约束

下面用一张表解释：

create table people（

表中数据的增删改查

添加数据：字符串类型或者日子类型要用单引号包起来
insert into people values（1，'tom','nan'...）//这里数据与表格顺序一一对应填入

修改语句：
update people set salary=4000,job='BOSS' where username='张三';//
修改张三的薪水和工作为4000，BOSS

删除数据：
delete from table where...（delete删除是一条一条删除的）
truncate table；删除表中所有数据（先删除整个表，在建立一个新的空的表，效率高）

查询语句：
  select * from table;(查询所有)
select name， job from table(显示查询字段名)
select distinct 字段名 from 表名；（过滤重复的数据，相同的只显示一次）

查询根据条件有很多中写法：
1.别名as：select name,math+english+chinese as sum from student;
  查询学生的姓名和总成绩，
2.常用符号一般在where后用来判断：
  < >  <= >= = <>不等于
3.like的查询范围
  like '%张'；  结果是XXX张
  like '张%'；  结果是张XXX
  LIKE '%张%';  结果是XXX张XXX
4.查询范围between...and..
  select *from student where math between 80 and 90;
5.查询范围in-在给定的范围
  select *from student where math in(65，76，97);
6.根据排序查询order by asc(升序)/desc(降序)
  select *from student where name like '%张' order by math desc
7.统计个数
  select count(*) from student where...;
8.求总和
  select sum(math) from student;
9.求平均
  select sum(math)/count(*) from student where...；
10.求平均，最大最小值
  select avg\max\min(math) from...
11.分组group by..having ... order by
  select product(*),sum(price) from orders group by product having 
  sum(price)>100 order by sum(price) desc;

事物

事物就是一个事情，组成这个事情可能有多个单元，要求这些单元要么全部成功，要么全部部成功

事物的存在，可以保证数据的完整性。

事物的特性

1.原子性：事物是一个不可分割的工作单位，事物的操作要么全部发生，要么都不发生。
2.一致性：事物前后的完整性必须保持一致。
3.隔离性：指多个用户访问数据库时，一个用户的事物不能被其他用户的事物所干扰
4.持久性：事物一旦被提交，对数据库的改变是永久的

事物的隔离级别

如果不考虑事物的隔离性会发生什么问题？
1.脏读：一个事物读取到另一个事物未提交的数据。
2.不可重复读：俩次读取数据不一样（读提交数据）-update
3.虚读：俩次读取数据不一样（读提交数据）-insert

隔离级别：
read uncommitted 什么问题也解决不了
read commited     可以解决脏读，其他解决不了
 Repeatable rread commitedead     可以解决脏读，不可重复读，不能解决虚读
Serializable     全部可以解决

Mysql默认隔离级别            Repeatable read    
oracle默认隔离级别        read commited
